Intent

sealed class Intent

Represents a user action or system event that should change the drawing state.

Intent is the input to the MVI (Model-View-Intent) pattern. All state changes flow through this sealed class:

User Action → Intent → Reducer → New State → UI Update
Content copied to clipboard

Intent instances are immutable data classes that capture the parameters of what the user wants to do. The io.ak1.drawbox.presentation.reducer.Reducer interprets these intents and produces new state.

EraseAt
Link copied to clipboard
data class EraseAt(val point: Offset, val radius: Float) : Intent

Delete every element whose body intersects a disk of radius around point (world space). Snapshots history lazily on the first hit within the current erase session (see BeginErase). Subsequent hits inside the same session mutate elements in place without snapshotting, so a single undo reverts the entire sweep. No-op when no element is hit — no snapshot is pushed and no State copy is emitted.
