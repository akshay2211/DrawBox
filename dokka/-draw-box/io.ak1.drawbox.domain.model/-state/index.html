State

Represents the complete immutable state of the drawing canvas.

All drawing operations produce a new State instance rather than mutating the existing one, following functional programming principles. This ensures:

  • Clear state history for undo/redo

  • Predictable state changes

  • Easy debugging and testing

toolMemory
Link copied to clipboard
val toolMemory: Map<Mode, ToolSettings>

Per-tool memory: each drawing mode (PEN, RECTANGLE, CIRCLE, TRIANGLE, ARROW, LINE, TEXT) snapshots its own color / width / style / fill / corner / text settings. On Intent.SetMode the outgoing mode's current top-level fields are captured here, and the incoming mode's saved settings are restored to the top-level fields. Non-drawing modes (SELECT, ERASER, PAN) don't participate — they pass through settings untouched. Users get the "each pen remembers its own color" behavior Samsung Notes / OPPO Notes use.
